# 时间与天气系统说明

本文档对应项目主文件 `index.html` 中的实现，说明**游戏内时间**与**天气**的数据模型、更新顺序、判定规则，以及二者如何互相影响。

---

## 一、总览

| 子系统 | 作用 |
|--------|------|
| **时间系统** | 用单一标量 `gameTime`（0～1）表示一昼夜，驱动太阳/月亮位置、主光与环境光、星空透明度，并在**晴天**下驱动天空背景色与夜间薄雾。 |
| **天气系统** | 状态机管理四种天气（晴/雨/雪/雾），控制雾、天空固定色、雨雪粒子；按**当前游戏时段**与**上一段天气**加权随机决定下一段天气；天气变化带平滑过渡。 |

**二者关联的核心原则：**

- **晴天（`clear`）且处于稳定期（`active`）**：天空颜色、部分雾效由**时间**主导（`getSkyColor(gameTime)`）。
- **非晴天，或正在过渡（`transitioning`）**：天空与雾主要由**天气参数表**主导；过渡时会把「当前时刻的晴天天空色」作为 `clear` 端点与目标天气色做插值。
- **随机下一天气**时，会读取**当前 `gameTime` 换算成的小时**，映射到四个时段之一，再查概率表与偏置表——这是时间影响天气的**唯一逻辑链路**（天气不会改变时间流逝）。

---

## 二、时间系统

### 2.1 核心变量

- **`gameTime`**：浮点数，范围经归一化后落在 **\[0, 1)**。表示一昼夜中的位置。
- **`DAY_DURATION`**：**300 秒（5 分钟）** 现实时间 = 游戏内完整一天。  
  每帧：`gameTime = (gameTime + deltaTime / DAY_DURATION) % 1.0`

### 2.2 与 24 小时制的对应关系

内部换算：**小时（带小数）= `gameTime * 24`**，再拆成 `HH:MM` 显示在 HUD 与控制台。

约定性锚点（与太阳角度公式一致）：

| `gameTime` | 含义（设计意图） |
|------------|------------------|
| `0` | 子夜 |
| `0.25` | 日出（太阳角度公式以此为相位基准） |
| `0.5` | 正午 |
| `0.75` | 日落 |

**注意：** 天空关键帧 `SKY_STOPS` 与上述锚点配合使用，日出/日落附近会有橙红色渐变。

### 2.3 太阳、月亮与星空

- 太阳角度：`sunAngle = (gameTime - 0.25) * 2π`  
  太阳位置在半径约 **600** 的圆周上运动（`sunY = sin(sunAngle)*R`，`sunZ = cos(sunAngle)*R`，并带有固定偏移 **x=80**）。
- 月亮位置与太阳**对径**（约在太阳反方向），形成简单昼夜天体布局。
- **太阳高度因子**：`sunH = sin(sunAngle)`，范围约 \[-1, 1\]。  
  **白昼因子**：`dayFactor = max(0, sunH)`，用于光照强度（仅取日照上半周）。

可见性规则（节选）：

- `sunH > -0.12` 时显示太阳（及光晕）。
- `sunH < 0.12` 时显示月亮。
- 星空透明度随 `-sunH` 增大而升高（夜间更明显）。

### 2.4 光照

- **平行光（太阳）**：位置跟随太阳；强度约 `dayFactor * 0.9 + 0.05`；颜色在日出/日落时更偏橙（用 `sunsetBlend` 与 `sunH` 混合 RGB）。
- **环境光（半球光）**：强度约 `0.12 + 0.55 * dayFactor`，随白昼变亮。

### 2.5 天空颜色（`getSkyColor`）

- 数据：`SKY_STOPS` 为多个关键帧，每项为 `[gameTime, R, G, B]`（0～1 RGB）。
- 算法：找到 `gameTime` 落在哪两个关键帧之间，**线性插值** RGB。
- 覆盖从深夜、黎明、日出、上午、傍晚、日落再到深夜的渐变。

### 2.6 晴天下的雾（仅稳定晴天）

当 **`weatherType === 'clear'` 且 `weatherState === 'active'`**：

- 若 `dayFactor < 0.05`（接近深夜）：使用极淡的线性雾 `THREE.Fog`，颜色取当前天空色，近远裁 **400～1200**，营造夜间景深。
- 否则：**清除场景雾**（`scene.fog = null`）。

非晴天或过渡中不走这条分支，由天气逻辑接管雾。

### 2.7 玩家可调：控制台 `time` 指令

- 无参数：打印当前 `HH:MM` 与内部 `gameTime`。
- 支持预设：`dawn`/`日出`、`noon`/`正午`、`dusk`/`日落`、`midnight`/`午夜` 等（中英部分别名）。
- 支持 `HH:MM`。
- 纯数字：若在 \[0,1\] 视为内部 `gameTime`；否则视为 **0～23 小时** 转算为 `gameTime`。

设置后会将 `gameTime` 规范到 \[0,1)。

## 三、天气系统

### 3.1 天气类型

字符串枚举：**`clear`（晴）、`rain`（雨）、`snow`（雪）、`fog`（雾）**。

### 3.2 视觉参数表 `WEATHER_PARAMS`

每种天气包含（用于雾、天空、粒子透明度）：

- **雾**：`fogNear`、`fogFar`、颜色 `fogR/G/B`（线性雾插值用）。
- **粒子**：`rainOp`、`snowOp`（雨雪点材质透明度目标值）。
- **天空**：`skyR/G/B`；**晴天用 -1 表示「不由表定死，而用时间系统的 `getSkyColor`」**。

**稳定应用规则（`applyWeatherInstant`）：**

- **`fog`**：使用 **指数雾** `FogExp2`（密度约 0.100），背景为表定天空色（与线性雾参数不同，是刻意区分的大雾效果）。
- **`clear`**：清除由天气设置的雾；天空交给时间系统在 `updateTimeSystem` 里更新。
- **`rain` / `snow`**：线性雾 + 表定天空色；显示对应粒子并设目标透明度。

### 3.3 状态机

- **`weatherState`**：
  - **`active`**：当前天气已稳定生效，累计 `weatherTimer`。
  - **`transitioning`**：从 `fromWeather` 过渡到 `toWeather`，累计 `transitionTimer`。

- **`weatherDuration`**：当前**稳定阶段**应持续多久（秒），由 `genWeatherDuration(类型)` 在阶段开始时随机生成。
- **`TRANSITION_DUR`**：**20 秒**，天气切换的过渡时长。

**稳定阶段（`active`）逻辑概要：**

1. `weatherTimer += deltaTime`。
2. **地面湿润度 `groundWetness`**（0～1）：  
   - 雨天：约 **30 秒** 从 0 积到 1（`deltaTime/30`）。  
   - 非雨天：约 **120 秒** 从 1 回到 0（`deltaTime/120`）。  
   *当前代码中该变量仅被更新，未参与材质或其它系统读取，可视为预留或后续扩展接口。*
3. 若 `weatherTimer >= weatherDuration`：  
   - 调用 `pickNextWeather(当前 weatherType)` 得到 `next`。  
   - 若 `next === weatherType`：**同类型延续**——重新随机一段持续时间，计时清零，视觉上无切换。  
   - 否则：**`startWeatherTransition(next)`** 进入过渡。

**过渡阶段（`transitioning`）逻辑概要：**

1. `transitionTimer += deltaTime`，`t = min(1, transitionTimer / TRANSITION_DUR)`。
2. **`applyWeatherLerp(t)`**：对雾参数、天空色、雨雪透明度做 **smoothstep** 平滑插值。
3. `t` 到达 1：切回 `active`，`weatherType = toWeather`，`applyWeatherInstant(toWeather)`，并生成新的 `weatherDuration`。

### 3.4 持续时间 `WEATHER_DURATION`（秒，闭区间内均匀随机）

| 天气 | 最小～最大（秒） |
|------|------------------|
| clear | 300～360 |
| rain | 120～180 |
| snow | 120～180 |
| fog | 60～120 |

### 3.5 时段划分 `getTimeSlot()`（决定天气概率）

用 **`h = gameTime * 24`（小时）** 判定：

| 返回值 | 小时范围 | 说明 |
|--------|----------|------|
| `morning` | \[5, 11) | 早晨 |
| `noon` | \[11, 17) | 正午前后 |
| `evening` | \[17, 21) | 傍晚 |
| `night` | 其余 | 夜间（含 0～5 与 21～24） |

### 3.6 概率表 `WEATHER_PROBS`（会与偏置相乘后归一化）

表中为相对权重（不必和为 100，实现中会再归一）：

| 时段 | clear | rain | snow | fog |
|------|-------|------|------|-----|
| morning | 40 | 20 | 15 | 25 |
| noon | 60 | 20 | 10 | 10 |
| evening | 30 | 30 | 15 | 25 |
| night | 20 | 25 | 20 | 35 |

直观理解：**正午最晴、雾最少；夜间雾最多；傍晚雨略多。**

### 3.7 转移偏置 `WEATHER_BIAS`

在抽到下一天气时，每种**上一天气**对四种候选有**乘数**，使转换更连贯，例如：

- 雨后更容易续雨或转雾，略难直接大晴。
- 雪后易续雪或起雾。
- 雾后易续雾或转雨。
- 晴后略易维持晴。

**抽样算法 `pickNextWeather(prev)`：**

1. `slot = getTimeSlot()`，`base = WEATHER_PROBS[slot]`。
2. 对每种类型 `t`：`weight[t] = max(0.01, base[t] * (WEATHER_BIAS[prev][t] || 1))`。
3. 按权重做一轮 roulette 随机。

### 3.8 初始化

`initWeatherSystem()` 末尾：`pickNextWeather('clear')` 作为**初始天气**（即假装上一段是晴），再 `applyWeatherInstant`，因此开局天气已受**当前 `gameTime` 对应时段**影响。

### 3.9 雨雪粒子与跟随范围

- 雨量点数 **6000**，雪 **2000**。
- **`WEATHER_RANGE = 35`**：粒子在玩家水平面周围约 ±35 单位内循环重生，并随玩家移动，保证近身有降水效果。
- 雨下落速度约 **18 单位/秒**（Y 方向）；雪更慢并带正弦漂移。
- `updateWeather` 在 **`updateTimeSystem` 之前**于动画主循环中调用；粒子只要在过渡/显示需要时可见就会更新，避免过渡时“冻结”。

### 3.10 玩家可调

- **G 键**：按顺序循环 `clear → rain → snow → fog`，均通过 `startWeatherTransition` 带过渡。
- 控制台 **`weather` / `we`**：  
  - 无参数：说明用法。  
  - 带类型：调用 `setWeather`，同样走过渡。

## 四、时间 ↔ 天气：判定与优先级小结

1. **时间单向影响天气随机**  
   仅通过 `getTimeSlot()` → `WEATHER_PROBS` + `WEATHER_BIAS` 影响「下一阶段抽什么天气」，不反向修改 `gameTime`。

2. **谁控制天空与雾（渲染优先级）**  
   - **晴天 + active**：`updateTimeSystem` 里用 `getSkyColor(gameTime)` 设 `scene.background`，并按深夜条件设/清线性雾。  
   - **雨天/雪天 active**：`applyWeatherInstant` 设固定天空色 + 线性雾。  
   - **雾天 active**：指数雾 + 固定背景色。  
   - **transitioning**：`applyWeatherLerp` 中，凡 `skyR < 0` 的端点用**当前** `getSkyColor(gameTime)` 代入，使从晴切入恶劣天气时过渡自然；雾统一用线性雾插值（若从 `FogExp2` 切入过渡，会先转成线性雾再插值）。

3. **光照**  
   太阳/环境光强度与颜色主要由 **`gameTime` 推导的 `sunH` / `dayFactor`** 决定，**不因天气类型直接改强度**（天气通过雾和天空间接改变观感）。

4. **HUD**  
   `updateTimeHUD()` 同时显示 **时钟** 与 **当前 `weatherType` 图标文案**，在时间与天气变化时都会刷新。
